# Projet : assistant santé vocal Apple Watch + Voxtral

## Contexte
On a déjà un pipeline vocal fonctionnel dans `gm` (github.com/spideystreet/gm) :
- STT : Voxtral Transcribe → texte
- LLM : Mistral (streaming tokens)
- TTS : Voxtral Mini TTS (streaming SSE → PCM float32 → OutputStream sounddevice)
- Waveform viz dans le terminal (Rich Live, pixel aesthetic orange Mistral)
- Silence detection pour le recording
- Conversation multi-turn

Tu peux t'en inspirer fortement pour l'archi audio/voix. Le code est dans ~/Developer/git/gm.

## Ce qu'on veut construire
Un CLI Python qui :
1. Récupère les données santé Apple Watch (FC, SpO2, sommeil, pas, calories, stress/HRV) via Apple Shortcuts → POST JSON vers un endpoint local
2. Intègre ces données comme contexte dans un LLM Mistral avec un system prompt orienté santé
3. Permet une conversation vocale (Voxtral STT/TTS) où tu décris tes symptômes et l'assistant croise avec tes données réelles
4. Affiche les données santé pertinentes dans le terminal + waveform viz

## Stack
- Python, Mistral API (mistralai SDK), sounddevice, Rich
- Petit serveur HTTP local (FastAPI ou juste http.server) pour recevoir les données du Shortcut iOS
- SQLite ou JSON pour stocker l'historique des données santé
- Apple Shortcuts côté iPhone pour push les données HealthKit

## Archi cible
```
iPhone Shortcut (HealthKit) → POST JSON → health_server.py → SQLite
                                                                ↓
CLI : audio.py → voxtral.py (STT) → brain.py (LLM + contexte santé) → voxtral.py (TTS) → viz.py
```

## Contraintes
- Pas de secrets en dur, tout en env vars
- Code et comments en anglais
- Conventional commits
- DA terminal : pixel aesthetic Mistral (orange #ff7000, block chars, waveform animée)
- Le LLM ne doit JAMAIS poser de diagnostic médical, toujours orienter vers un professionnel. C'est un assistant de suivi, pas un médecin.
